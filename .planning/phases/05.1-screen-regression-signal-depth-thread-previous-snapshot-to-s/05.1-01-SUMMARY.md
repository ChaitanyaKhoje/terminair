## Decisions Made

- `get_previous_models()` returns only the four ModelState fields that RegressionAnalyzer reads from previous items — avoids re-running all expensive enrichment (Airflow, Snowflake) for historical snapshot data
- MockDataProvider grain drift chosen so both grain signal directions are covered: grain_removed (fct_revenue_daily loses region from current perspective) and grain_added (fct_risk_exposure gains entity_id from current perspective)
- ModelState required fields filled with empty defaults in StateAggregator.get_previous_models() — cleaner than making them Optional in the dataclass
