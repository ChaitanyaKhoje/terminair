Fuzzy match helper (from RESEARCH.md Pattern 4 — use difflib, no new deps):
```python
import difflib

def _fuzzy_match(task_id: str, node_names: list[str]) -> str | None:
    # Substring check first (handles run_fct_revenue_daily → fct_revenue_daily)
    for name in node_names:
        if name.lower() in task_id.lower() or task_id.lower() in name.lower():
            return name
    matches = difflib.get_close_matches(task_id, node_names, n=1, cutoff=0.6)
    return matches[0] if matches else None
```

**Module-level helper** `_fuzzy_match(task_id: str, node_names: list[str]) -> str | None`:
- Substring check first (case-insensitive): for each name in node_names, if name.lower() is in task_id.lower() OR task_id.lower() is in name.lower(), return the full node name string
- Fallback: `difflib.get_close_matches(task_id, node_names, n=1, cutoff=0.6)` — return matches[0] or None

**Class AirflowBridge:**

`__init__(self, connection: Connection) -> None`:
- `self._client = httpx.AsyncClient(base_url=str(connection.url), auth=build_auth(connection.auth), timeout=10.0)`
- Do NOT import Connection at module level (use TYPE_CHECKING guard to avoid circular import risk)
- No HTTP calls in __init__

`async def get_task_statuses(self, dag_names: list[str], node_names: list[str]) -> dict[str, tuple[str, str | None]]`:
- Result dict: `result: dict[str, tuple[str, str | None]] = {}`
- For each dag_id in dag_names:
  - Wrap entire dag loop body in try/except httpx.HTTPError — catch, log with `_log.warning`, raise AirflowBridgeError(sanitize_error(str(e))) from e
  - GET `/api/v1/dags/{dag_id}/dagRuns` with params `{"limit": 1, "order_by": "-execution_date"}` — use `.raise_for_status()`
  - Extract `runs = resp.json().get("dag_runs", [])`; if empty, continue
  - GET `/api/v1/dags/{dag_id}/dagRuns/{dag_run_id}/taskInstances`
  - For each ti in `resp.json().get("task_instances", [])`:
    - `task_id = ti["task_id"]`
    - `matched_name = _fuzzy_match(task_id, node_names)`
    - If matched: build full node_id `f"model.my_project.{matched_name}"` — NOTE: in real use, the caller should provide the full node_id → name mapping; for now construct a best-effort node_id this way
    - Actually: the caller passes `node_names` as a list of short names (e.g., "fct_revenue_daily"); the function should return `{matched_name: (ti.get("state", "unknown"), ti.get("hostname"))}` — using matched_name as key so StateAggregator can resolve. StateAggregator will map via manifest.get_all_node_ids().
    - Store: `result[matched_name] = (ti.get("state", "unknown"), ti.get("hostname"))`
- Return result
